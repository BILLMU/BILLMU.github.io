在神经网络中，激活函数、权重初始化方法和损失函数需要合理搭配，以确保模型训练的稳定性和效率。以下是常见的组合推荐：

1. Sigmoid 激活函数

  • 适用场景：二分类（单标签或多标签）、输出概率值
  • 数值初始化：
    • Xavier初始化：nn.init.xavier_uniform_ 或 nn.init.xavier_normal_
    • 原因：保持输入和输出的方差一致，缓解梯度消失
  • 损失函数：
    • 二分类：nn.BCELoss()（需配合 Sigmoid 输出）
    • 多标签分类：nn.BCEWithLogitsLoss()（集成 Sigmoid 和 BCE，数值更稳定）
  • 注意事项：
    • Sigmoid在输入值较大或较小时容易饱和，导致梯度消失
    • 不推荐用于隐藏层，通常只用于输出层

2. Softmax 激活函数

  • 适用场景：多分类（单标签，每个样本仅属于一类）
  • 数值初始化：
    • Xavier初始化：同上
  • 损失函数：
    • nn.CrossEntropyLoss()（集成 LogSoftmax 和 NLLLoss）
    • 输入要求：模型输出为未归一化的logits，无需手动添加Softmax
  • 注意事项：
    • 若手动使用 nn.Softmax，则需配合 nn.NLLLoss()
    • Softmax输出的概率和为1，表示类别分布

3. ReLU 激活函数

  • 适用场景：隐藏层（深度学习最常用）
  • 数值初始化：
    • Kaiming初始化：nn.init.kaiming_uniform_ 或 nn.init.kaiming_normal_
    • 参数：mode='fan_in'（默认），nonlinearity='relu'
    • 原因：针对ReLU的非对称性，保持方差一致性
  • 损失函数：
    • 分类：同Sigmoid/Softmax（取决于任务类型）
    • 回归：nn.MSELoss() 或 nn.L1Loss()
  • 注意事项：
    • 存在“死亡ReLU”问题（输入长期<0时梯度为0）
    • 可使用LeakyReLU、PReLU等变体缓解

4. Tanh 激活函数

  • 适用场景：隐藏层（输出范围[-1, 1]，零中心）
  • 数值初始化：
    • Xavier初始化：同上
  • 损失函数：
    • 同ReLU（取决于具体任务）
  • 注意事项：
    • 比Sigmoid更接近零中心，梯度消失问题稍轻
    • 仍存在饱和问题，不如ReLU高效

5. LeakyReLU/PReLU

  • 适用场景：替代ReLU，缓解“死亡ReLU”问题
  • 数值初始化：
    • Kaiming初始化：同上，但需指定 nonlinearity='leaky_relu'
    • 负斜率参数：LeakyReLU默认0.01，PReLU可学习该参数
  • 损失函数：
    • 同ReLU

6. Swish/SiLU

  • 适用场景：高性能网络（如EfficientNet）
  • 数值初始化：
    • Kaiming初始化：同上
  • 损失函数：
    • 同ReLU

7. GELU

  • 适用场景：Transformer等高端架构
  • 数值初始化：
    • Kaiming初始化：同上
  • 损失函数：
    • 同ReLU

关键原则

  1. 非线性激活：隐藏层避免使用线性激活（除非构建线性模型）
  2. 初始化匹配：
    • Sigmoid/Tanh → Xavier初始化
    • ReLU及其变体 → Kaiming初始化
  3. 损失函数选择：
    • 分类任务 → 交叉熵损失（BCE或CrossEntropy）
    • 回归任务 → MSE或L1损失
  4. 输出层设计：
    • 二分类/多标签 → Sigmoid + BCE
    • 多分类（单标签） → Softmax + CrossEntropy（或直接用CrossEntropyLoss）
