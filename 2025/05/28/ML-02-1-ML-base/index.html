2.机器学习基本理论

术语

数据集（Data Set）：多条记录的集合。

  • 训练集（Training Set）：用于训练模型的数据。
  • 验证集（Validation Set）：用于调节超参数的数据。
  • 测试集（Test Set）：用于评估模型性能的数据。

样本（Sample）：数据集中的一条记录是关于一个事件或对象的描述，称为一个样本。
特征（Feature）：数据集中一列反映事件或对象在某方面的表现或性质的事项，称为特征或属性。
特征向量（Feature Vector）：将样本的所有特征表示为向量的形式，输入到模型中。
标签（Label）：监督学习中每个样本的结果信息，也称作目标值（target）。
模型（Model）：一个机器学习算法与训练后的参数集合，用于进行预测或分类。
训练 (Training)：通过模型得到参数的过程
参数（Parameter）：模型通过训练学习到的值，例如线性回归中的权重和偏置。
超参数（Hyper Parameter）：由用户设置的参数，不能通过训练自动学习，例如学习率α\alphaα、正则化系数λ\lambdaλ等。

2.1 机器学习三要素

机器学习方法 = 模型 + 策略 + 算法

  • 模型（model）：总结数据的内在规律，用数学语言描述的参数系统
  • 策略（strategy）：选取最优模型的评价准则
  • 算法（algorithm）：选取最优模型的具体方法

2.2 机器学习方法分类

机器学习的方法种类繁多，并不存在一个统一的理论体系能够涵盖所有内容。从不同的角度，可以将机器学习的方法进行不同的分类：

  • 通常分类：按照有无监督，机器学习可以分为 有监督学习、无监督学习 和 半监督学习，除此之外还有 强化学习。
    • 有监督学习：提供数据对应结果的机器学习过程
    • 无监督学习： 提供数据当不提供数据对应结果的学习过程
    • 强化学习：多应用于行为主义，通过与环境交互，提供奖惩机制来获取延迟返回并进行改进的学习过程。
  • 按模型分类：根据模型性质，可以分为概率模型/非概率模型，线性/非线性模型等。
  • 按学习技巧分类：根据算法基于的技巧，可以分为贝叶斯学习、核方法等。

2.2.1 无监督学习

  • 数据特点：输入数据没有标注的目标变量 ，不像监督学习有明确的 “输入 - 输出” 对应关系。比如一堆用户消费记录数据，没有事先标记哪些是 “高消费用户”“低消费用户” 等类别。
  • 学习方式：从无标记数据中挖掘内在结构和模式 。像通过分析用户消费记录的金额、频次等特征，把相似消费行为的用户归为一类，这就是聚类过程；或者估计数据在特征空间中的分布密度，判断哪些区域数据更集中。
  • 反馈机制：不依赖外部反馈来调整模型 。不是像监督学习那样根据预测值和真实值的误差去优化，而是自主发现数据共性特征。新数据来了，对比是否有已发现的共性特征，判断归属于哪类或处于何种分布位置。
  • 核心应用
    • 聚类分析：比如电商根据用户购物偏好、频次等将用户聚类，针对性营销；文档根据主题、关键词等聚类，方便信息管理。
    • 密度估计：在异常检测中，通过估计正常数据的分布密度，找出偏离正常密度区域的数据点，视为异常。

2.2.2 监督学习

定义

监督学习通过利用包含输入和对应输出（标签 ）的训练数据构建数学模型 。训练数据中的每个样本都有明确的输入特征和对应的输出值，模型从这些数据中学习规律，以对新输入进行预测。比如用大量标注了 “垃圾邮件” 或 “正常邮件” 的邮件文本训练模型，让模型学习区分新邮件类别。

主要任务类型
  • 分类：当输出是有限个离散值时采用分类算法。像判断图片是猫、狗还是其他动物，类别是离散且有限的 。常见分类算法有决策树、支持向量机、逻辑回归等。
  • 回归：输出为连续数值时使用回归算法 。例如根据房屋的面积、房龄等特征预测房价，房价可以在一定范围内取任意连续值。常用回归算法有线性回归、岭回归、决策树回归等。
相似度学习

它是监督学习的一种延伸，借助相似度函数度量对象间相似或关联程度 。在推荐系统中，通过计算用户或物品间相似度，为用户推荐相似物品；在人脸识别里，计算人脸图像间相似度判断是否为同一人。 它融合了分类和回归的思想，能处理更复杂的相似性度量与预测任务。

流程解读
  1. 训练数据输入：(x 1 ,y 1 ),(x 2 ,y 2 ),...,(x N ,y N )(x~1~, y~1~), (x~2~, y~2~),..., (x~N~, y~N~)(x 1 ,y 1 ),(x 2 ,y 2 ),...,(x N ,y N )代表包含 N 个样本的训练数据集 ，其中x i x~i~x i 是输入特征向量，y i y~i~y i 是对应的标签（真实值 ） 。这些数据输入到 “学习系统”。
  2. 模型训练：学习系统基于输入的训练数据进行学习，调整模型参数，最终得到一个模型。该模型可以表示为Y=f^(X)Y = \hat{f}(X)Y=f^​(X)，这是一个从输入 X 到输出 Y 的映射函数；也可以表示为P^(Y∣X)\hat{P}(Y|X)P^(Y∣X)，即给定输入 X 时输出 Y 的条件概率估计 。
  3. 预测阶段：对于新的输入数据xN+1x_{N + 1}xN+1​，将其输入到 “预测系统”，预测系统利用训练好的模型对其进行预测，输出预测值yN+1y_{N + 1}yN+1​
示例说明

比如在房价预测中 ：

-(x 1 ,y 1 ),(x 2 ,y 2 ),...,(x N ,y N )(x~1~, y~1~), (x~2~, y~2~), ..., (x~N~, y~N~)(x 1 ,y 1 ),(x 2 ,y 2 ),...,(x N ,y N ) 里的x i x~i~x i 可能包含房屋面积、房间数量、房龄等特征，y i y~i~y i 就是对应的房屋实际价格。

  • 学习系统通过分析这些训练数据，构建出房价预测模型。
  • 当有新房屋的特征xN+1x_{N + 1}xN+1​时，预测系统使用模型给出该房屋价格的预测值yN+1y_{N + 1}yN+1​。

2.3 机器学习方法汇总

2.3.1 有监督学习

利用带标签数据学习，预测未知数据标签。

  • 线性模型：

    • 线性回归：通过最小二乘法寻找最佳拟合直线（平面等），用于预测连续值 。
    • [Lasso 回归](2.6_模型求解算法.md#2.7.1 L1正则化（Lasso回归）)：在线性回归基础上加入 L1 正则化，可进行特征选择，防止过拟合。
    • [岭回归](2.6_模型求解算法.md#2.7.2 L2正则化（Ridge回归，岭回归）)：引入 L2 正则化，给系数加上惩罚项，降低模型复杂度。
    • 线性判别：用于分类，找最佳线性投影，使不同类别数据尽可能分开。
    • 逻辑回归：用于二分类或多分类，基于线性回归，通过 Sigmoid 函数将线性结果映射到 0 - 1 之间表示概率。

  • K 近邻：给定测试样本，找训练集中 K 个距离最近样本，根据其标签投票或加权平均确定测试样本标签。

  • [决策树](07_其他监督学习算法（了解）.md#7.2 决策树)：基于特征对样本进行递归划分，形成树形结构，每个节点是特征判断，分支是判断结果，叶节点是类别或值。

  • [朴素贝叶斯](07_其他监督学习算法（了解）.md#7.1 朴素贝叶斯法)：基于贝叶斯定理，假设特征间相互独立，计算样本属于不同类别的概率。

  • [支持向量机](07_其他监督学习算法（了解）.md#7.3 支持向量机 )：寻找最优超平面，使不同类别样本间隔最大化，可处理线性和非线性分类问题。

  • 神经网络：由大量神经元连接组成，通过反向传播算法更新权重，学习输入和输出间复杂映射关系。

  • 集成学习：组合多个弱学习器提升性能。

    • Boosting：串行训练弱学习器，每个学习器纠正前一个的错误，如 AdaBoost、梯度提升树 。

      • [AdaBoost](07_其他监督学习算法（了解）.md#7.4.1 AdaBoost（Boosting）)：不断调整样本权重，让后续学习器关注难分类样本。
      • 梯度提升树：基于梯度下降原理，每次拟合残差。
      • XGBoost：对梯度提升树优化，速度快、可扩展性好。
      • LightGBM：采用直方图算法等优化，减少内存占用和计算量。

    • Bagging：并行训练多个学习器，最后综合结果，[随机森林](07_其他监督学习算法（了解）.md#7.4.2 随机森林（Bagging）)是典型代表，通过对样本和特征随机采样构建多棵决策树并投票。

    • Stacking：训练多个不同类型的个体学习器，之后使用一个元模型综合多个个体学习器的预测。灵活性强，能结合多种模型的优势。

2.3.2 无监督学习

从无标签数据找内在结构模式。

  • [聚类](08_无监督学习.md#8.1 聚类)：将数据对象分组，组内相似性高，组间差异大。

    • K-means：指定聚类数 K，通过迭代更新质心，使样本到所属质心距离和最小。
    • 高斯混合聚类：假设数据由多个高斯分布混合生成，估计分布参数实现聚类。
    • 密度聚类：基于数据点密度，密度相连区域划分为一类，能发现任意形状簇，如 DBSCAN。
    • 层次聚类：分凝聚式（自底向上）和分裂式（自顶向下），通过计算样本间距离逐步合并或分裂簇。
    • 谱聚类：根据样本间相似度构建图，基于图的拉普拉斯矩阵特征值和特征向量聚类。

  • [降维](08_无监督学习.md#8.2 降维（了解）)：减少数据特征维度，保留主要信息。

    • 主成分分析：通过线性变换将原始数据投影到新特征空间，使方差最大化，提取主成分。
    • 奇异值分解：矩阵分解方法，用于降维、数据压缩等。
    • t-SNE：非线性降维技术，用于高维数据可视化，保持数据局部结构。
    • 自编码器：神经网络结构，通过压缩和解压缩数据学习数据特征表示，实现降维。

2.3.3 半监督学习

结合少量有标签和大量无标签数据学习。

  • 半监督 SVM：在 SVM 基础上，利用无标签数据调整分类超平面。
  • 图半监督学习：将数据表示为图结构，节点是样本，边表示样本间关系，基于图的平滑性假设传播标签。
  • 生成模型：假设数据由某种概率模型生成，结合有标签和无标签数据估计模型参数，如高斯混合模型扩展到半监督场景。
  • 协同训练：利用数据不同视图，在每个视图上训练模型，相互提供伪标签扩充有标签数据集。
  • 概率模型：如 EM 算法（期望最大化算法），用于含有隐变量的概率模型参数估计；还有朴素贝叶斯、贝叶斯网络等在半监督场景的应用，以及隐马尔可夫模型等。

2.3.4 强化学习

智能体在环境中通过试错与环境交互，以最大化长期累积奖励为目标学习最优策略。

  • 动态规划：将复杂问题分解为子问题，利用子问题解求解原问题，用于求解马尔可夫决策过程。
  • 蒙特卡洛树搜索：通过模拟多次游戏或决策过程，构建搜索树，选择最优行动。
  • Q 学习：基于值函数估计，学习状态 - 动作值函数 Q，选择使 Q 值最大的动作。
  • 策略梯度算法：直接对策略参数进行优化，根据梯度上升方向更新策略参数以最大化奖励。
  • 模仿学习：智能体通过模仿专家示范行为学习策略。

2.3.5 概率模型

EM 算法（期望最大化算法 ）
  • 原理：是一种迭代算法，用于含有隐变量的概率模型参数的极大似然估计。每次迭代包含两步，E 步（期望步 ）利用当前估计的参数值计算对数似然的期望值；M 步（最大化步 ）寻找能使 E 步产生的似然期望最大化的参数值。不断重复这两步，直到收敛。
  • 应用：广泛用于聚类（如高斯混合模型的参数估计 ）、缺失数据处理等场景。例如在语音识别中，对含有隐状态的声学模型进行参数估计。
朴素贝叶斯
  • 原理：基于贝叶斯定理，假设所有特征相互独立。通过计算每个类别在给定特征下的后验概率，选择后验概率最大的类别作为预测结果。公式为P(C∣X)=P(X∣C)P(C)P(X)P(C|X)=\frac{P(X|C)P(C)}{P(X)}P(C∣X)=P(X)P(X∣C)P(C)​，其中 C 是类别，X 是特征向量。
  • 应用：常用于文本分类（如垃圾邮件过滤 ）、情感分析等领域，因其简单高效，在数据量较大时表现良好。
贝叶斯网络
  • 原理：是一种有向无环图模型，节点代表随机变量，边代表变量间的依赖关系，每个节点都有一个条件概率表描述其与父节点的概率关系。通过联合概率分布表示变量间的依赖和独立关系，进行概率推理。
  • 应用：在专家系统、故障诊断、生物信息学等领域有应用，可处理不确定性知识，进行因果分析和预测。
隐马尔可夫模型（HMM ）
  • 原理：是一种统计模型，描述由隐藏的马尔可夫链随机生成不可观测的状态随机序列，再由各个状态生成观测序列的过程。包含状态转移概率、观测概率和初始状态概率三个要素，通过前向 - 后向算法等进行参数估计和预测。
  • 应用：在语音识别、自然语言处理（词性标注等 ）、生物序列分析（DNA 序列分析 ）等领域广泛应用。
马尔可夫随机场
  • 原理：是一种无向图模型，节点表示随机变量，边表示变量间的概率依赖关系。满足马尔可夫性质，即一个节点的状态只依赖于其相邻节点的状态。通过定义势函数来描述节点间的相互作用，进而定义联合概率分布。
  • 应用：在图像分割、图像去噪、立体视觉等计算机视觉领域，以及自然语言处理的句法分析等方面有应用。
条件随机场
  • 原理：是给定一组输入随机变量条件下另一组输出随机变量的条件概率分布模型，常用于标记序列数据。是一种无向图模型，通过定义特征函数和权重来构建条件概率分布，目标是最大化标记序列的条件概率。
  • 应用：在自然语言处理的命名实体识别、词性标注、语义角色标注等任务中广泛应用。
马尔科夫链蒙特卡洛方法（MCMC ）
  • 原理：通过构造一个马尔可夫链，使其平稳分布就是要采样的目标分布，在马尔可夫链上进行随机游走，经过一定步数达到平稳状态后，得到的样本近似服从目标分布。常用的算法有 Metropolis - Hastings 算法和吉布斯采样。
  • 应用：在贝叶斯统计推断、复杂概率模型的参数估计和后验分布计算等方面应用广泛，用于处理高维、复杂的概率分布采样问题。
隐狄利克雷分配（LDA ）
  • 原理：是一种主题模型，假设文档是由多个主题混合而成，每个主题是词的概率分布。通过对大量文档的词频信息进行分析，挖掘出文档集合中的潜在主题结构，是一种无监督学习方法。
  • 应用：在文本挖掘领域用于文本主题建模、文档分类、信息检索等，可帮助理解文档集合的主题分布情况。
变分自编码器（VAE ）
  • 原理：是一种生成式模型，结合了自编码器和变分推断。通过引入潜在变量和变分近似，将输入数据编码到潜在空间，再从潜在空间解码重构数据。利用变分下界最大化来训练模型，学习数据的分布表示。
  • 应用：在图像生成、图像修复、数据降维等领域有应用，可生成与训练数据相似的新数据样本。

2.4 建模流程（项目流程）

1. 收集数据

从各种来源获取用于训练和测试的数据集 ，如传感器采集、网络爬虫、数据库抽取等。数据要具有代表性，全面覆盖实际问题涉及的不同情况和特征，为后续模型学习提供丰富信息。例如图像识别项目，需收集多样的图像数据，包含不同光照、角度、姿态下的目标图像。

2. 数据清洗

对原始数据处理，去除噪声、重复值、缺失值等 “脏数据” 。比如填充或删除缺失值（根据情况用均值、中位数填充数值型缺失值 ）；识别并剔除重复样本；处理错误格式数据等，提升数据质量，避免干扰模型训练。

3. 特征工程

  • 特征提取：从原始数据中提取有价值特征，如文本数据提取词频、TF - IDF 等特征；图像数据提取边缘、纹理等特征。
  • 特征转换：对特征进行数学变换，如归一化（将数值特征缩放到 0 - 1 或特定范围 ）、标准化（转化为均值为 0，方差为 1 的数据 ），使数据更适合模型训练。
  • 特征选择：从众多特征中挑选最具代表性、与目标相关性高的特征，减少冗余特征，降低计算量，提高模型性能，方法有过滤法、包装法、嵌入法等。

4. 选择算法

即选择机器学习方法。依据任务类型（分类、回归、聚类等 ）和数据特征（数据规模、分布、特征维度等 ）选择合适算法。分类任务可选决策树、支持向量机等；回归任务可考虑线性回归、随机森林回归等；聚类任务用 K - means、DBSCAN 等。同时兼顾算法的可解释性、计算资源需求等因素。

5. 模型训练

将清洗和特征工程处理后的数据分为训练集和验证集（有时还会分测试集 ），用训练集数据训练模型，模型通过学习数据中的模式和规律，调整内部参数（如神经网络的权重 ），构建输入到输出的映射关系。

6. 模型评估

用测试集数据评估训练好的模型性能 。不同任务有不同评估指标，分类任务常用准确率、精确率、召回率、F1 值等；回归任务用均方误差、平均绝对误差等。通过评估判断模型是否过拟合或欠拟合，是否达到预期性能目标。

7. 模型优化

若模型评估效果不佳，进行优化调整。可调整模型超参数（如决策树的最大深度、神经网络的学习率 ）；重新进行特征工程；尝试集成学习方法融合多个模型等，提升模型性能。

8. 模型部署

将优化好的模型部署到实际生产环境，如网页应用、移动应用、服务器后台等，使其能实时处理新数据并给出预测结果，同时持续监控模型表现，及时发现性能下降等问题并处理。

2.5 特征工程

特征工程（Feature Engineering）是机器学习过程中非常重要的一步，指的是通过对原始数据的处理、转换和构造，生成新的特征或选择有效的特征，从而提高模型的性能。简单来说，特征工程是将原始数据转换为可以更好地表示问题的特征形式，帮助模型更好地理解和学习数据中的规律。优秀的特征工程可以显著提高模型的表现；反之，忽视特征工程可能导致模型性能欠佳。

实际上，特征工程是一个迭代过程。特征工程取决于具体情境。它需要大量的数据分析和领域知识。其中的原因在于，特征的有效编码可由所用的模型类型、预测变量与输出之间的关系以及模型要解决的问题来确定。在此基础上，辅以不同类型的数据集（如文本与图像）则可能更适合不同的特征工程技术。因此，要具体说明如何在给定的机器学习算法中最好地实施特征工程可能并非易事。

2.5.1 特征工程的内容

1）特征选择

从原始特征中挑选出与目标变量关系最密切的特征，剔除冗余、无关或噪声特征。这样可以减少模型的复杂度、加速训练过程、并减少过拟合[^1]的风险。特征选择不会创建新特征，也不会改变数据结构。

（1）过滤法（Filter Method）

​ 基于统计测试（如卡方检验、相关系数、信息增益等）来评估特征与目标变量之间的关系，选择最相关的特征。

（2）包裹法（Wrapper Method）

​ 使用模型（如递归特征消除 RFE）来评估特征的重要性，并根据模型的表现进行特征选择。

（3）嵌入法（Embedded Method）

​ 使用模型本身的特征选择机制（如决策树的特征重要性，L1正则化的特征选择）来选择最重要的特征。

2）特征转换

对数据进行数学或统计处理，使其变得更加适合模型的输入要求。

（1）[归一化（Normalization）](3_KNN算法.md#3.1 归一化)

​ 将特征缩放到特定的范围（通常是0到1之间）。适用于对尺度敏感的模型（如KNN、SVM）。

（2）[标准化（Standardization）](3_KNN算法.md#3.2 标准化 )

​ 通过减去均值并除以标准差，使特征的分布具有均值0，标准差1。

标准化是将数据转换为均值为 0，标准差为 1 的分布。对于给定的数据集X={x1,x2,⋯ ,xn}X = \{x_1,x_2,\cdots,x_n\}X={x1​,x2​,⋯,xn​}，其中每个xix_ixi​是一个特征向量。标准化后的特征值xij′x_{ij}'xij′​可以通过以下公式计算：

xij′=xij−μjσjx_{ij}'=\frac{x_{ij}-\mu_j}{\sigma_j} xij′​=σj​xij​−μj​​

这里，xijx_{ij}xij​是第 i 个样本的第 j 个特征值，μj\mu_jμj​是第 j 个特征的均值，σj\sigma_jσj​是第 j 个特征的标准差。

（3）对数变换

​ 对于有偏态的分布（如收入、价格等），对数变换可以将其转化为更接近正态分布的形式。

（4）类别变量的编码

  • 独热编码（One-Hot Encoding）：将类别型变量转换为二进制列，常用于无序类别特征。
  • 标签编码（Label Encoding）：将类别型变量映射为整数，常用于有序类别特征。
  • 目标编码（Target Encoding)：将类别变量的每个类别替换为其对应目标变量的平均值或其他统计量。
  • 频率编码（Frequency Encoding）：*将类别变量的每个类别替换为该类别在数据集中的出现频率。
3）特征构造

特征构造是基于现有的特征创造出新的、更有代表性的特征。通过组合、转换、或者聚合现有的特征，形成能够更好反映数据规律的特征。

（1）交互特征

​ 将两个特征组合起来，形成新的特征。例如，两个特征的乘积、和或差等。
​ 例如，将年龄与收入结合创建新的特征，可能能更好地反映某些模式。

（2）统计特征

​ 从原始特征中提取统计值，例如求某个时间窗口的平均值、最大值、最小值、标准差等。
​ 例如，在时间序列数据中，你可以从原始数据中提取每个小时、每日的平均值。

（3）日期和时间特征

​ 从日期时间数据中提取如星期几、月份、年份、季度等特征。
​ 例如，将“2000-01-01”转换为“星期几”、“是否节假日”、“月初或月末”等特征。

4）特征降维

当数据集的特征数量非常大时，特征降维可以帮助减少计算复杂度并避免过拟合。通过降维方法，可以在保持数据本质的情况下减少特征的数量。

（1）[主成分分析（PCA）](08_无监督学习.md#8.2.2 主成分分析)

​ 通过线性变换将原始特征映射到一个新的空间，使得新的特征（主成分）尽可能地保留数据的方差。

（2）线性判别分析（LDA）

​ 一种监督学习的降维方法，也可用于分类，通过最大化类间距离与类内距离的比率来降维。

（3）t-SNE（t-Distributed Stochastic Neighbor Embedding，t分布随机近邻嵌入）

​ 一种非线性的降维技术，特别适合可视化高维数据。

（4）自编码器（Auto Encoder）

​ 一种神经网络模型，通过压缩编码器来实现数据的降维。

2.5.2 常用方法

对于一个模型来说，有些特征可能很关键，而有些特征可能用处不大。

例如：

  • 某个特征取值较接近，变化很小，可能与结果无关。
  • 某几个特征相关性较高，可能包含冗余信息。

因此，特征选择 在特征工程中是最基本、也最常见的操作。

另外，在训练模型时有时也会遇到维度灾难，即特征数量过多。我们希望能在确保不丢失重要特征的前提下减少维度的数量，来降低训练模型的难度。所以在特征工程中，也经常会用到 特征降维 方法。

1）低方差过滤法

对于特征的选择，可以直接基于方差来判断，这是最简单的。低方差的特征意味着该特征的所有样本值几乎相同，对预测影响极小，可以将其去掉。

3）主成分分析（PCA）

PCA 在不需要事先人为指定哪些特征重要的情况下，通过对数据的内在结构进行分析，基于特征值和特征向量等计算，自动筛选出能够最大程度保留原始数据信息的主要特征（主成分）

该机器算法主要用于创建与筛选特征，即在不知道特征值的情况下，使用PCA筛选对n维数据筛选出期待数量的主要特征值

主成分分析（Principal Component Analysis，PCA）是一种常用的降维技术，通过线性变换将高维数据投影到低维空间，同时保留数据的主要变化模式。

KNN算法介绍 K近邻算法（K-Nearest Neighbors，K·NN）是一种基本的分类与回归方法，属于监督学习算法。其核心思想是通过计算给定样本与数据集中所有样本的距离，找到距离最近的K个样本，然后根据这K个样本的类别或值来预测当前样本的类别或值。 1.1 工作原理 计算距离：计算待分类样本与训练集中每个样本的距离。 选择K个近邻：根据计算的距离，选择距离最近的K个样本。 投票或平均： 分类任务：统计K个近邻各类别的数量，将待分类样本归为数量最多的类别。 回归任务：取K个近邻的平均值作为预测结果。 1.2 关键参数 距离度量方法：选择合适的距离度量方法，常见的有欧氏距离、曼哈顿距离、切比雪夫距离、闵可夫斯基距离等。 K值：K值的选择对结果影响很大。K值过小容易过拟合，K值过大则可能欠拟合。 1.3 优缺点 KNN优点： 简单直观，易于理解和实现。 无需训练过程，直接利用训练数据进行预测。 KNN缺点： 计算量大，尤其是训练集较大时。 对噪声数据较敏感。 1.4 API使用 分类 123import numpy as npfrom...
